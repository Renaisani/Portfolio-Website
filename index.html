Porfolio Website(this)

My goal with this website is to have an easily accessible place to display my most up to date resume and projects for potential employers to review. I set up the design with mobile styling along with layout concepts pulled from a mobile UI art page on Instagram, followed by applying two color palettes copied from colorhunt.co. Next I built a simple HTML structure for each section of the website, and proceeded to add all the styling to each of the sections. Since my initial design is meant for mobile, I used a Google Chrome plugin called Mobile Simulator for testing how designs would look on a handheld device. After finishing the mobile styles, I connected the frontend with a backend express server made with node.js. Lastly, when it came to deploying the app, from my experience using Heroku, Digital Ocean, and Railway, the latter is by far the simplest deployment option and was my option of choice.

Tech used:

HTML, SCSS, Javascript, Node.js, Express.js, Railway

Weather API app

I took on this project so that I could lean to build an app that could connect to an API and then implement that data into a product. I chose to use the open weather API for this project which allowed me to experience reading API documentation to understand how to fetch the data and then connect that data to HTML elements. I accomplished this by making HTTP requests to pull the JSON data from the API. From there I used Body Parser to extract the necessary data from the JSON object and saved them into variables that could be inserted into HTML elements.

Tech used:

HTML, Open Weather API, Node.js, Express.js, Railway
